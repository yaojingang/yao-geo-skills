# Quality Gates

## Goal

即便这个 skill 完全开源，也要保证它输出的不是空泛模板，而是有证据、有个性化、有边界的可交付方案。

## Gate 1: Evidence Integrity

- 所有关键判断都必须能回到证据表
- 事实与推断必须分开写
- “最新”必须带绝对日期
- 证据不足时要降级置信度，而不是补想象

## Gate 2: Personalization

- 必须识别业务类型
- 必须识别国内 / 海外 / 混合 GEO 场景
- 必须描述典型转化路径
- 必须说明现有承接资产与站点能力
- 方案必须体现企业差异，而不是同一套动作照抄

## Gate 3: Attribution Structure

- 必须同时输出 `直接效果` 与 `间接效果`
- 必须解释 `品牌层价值` 与转化层价值的关系
- 必须包含 `效果追踪方法与原理说明`
- 必须包含 `监测效果边界说明`
- 必须给出指标定义、取数来源、更新频率
- 必须说明数据解释边界
- 必须给出优先级路线图

说明：

- 如果使用 `20%~30%` 之类的可观测占比，必须写成保守规划假设或边界提示
- 不能把该比例写成所有行业通用的固定事实

## Gate 4: Open-Source Safety

- 不引用任何私有系统、私有 CLI、私有 API、未授权文档
- 不在 examples、assets、reports 中保留真实公司私密信息
- 不保留绝对路径、账号名、token、cookie、内网域名
- 示例应使用公开可理解或合成的样例

## Gate 5: Deliverable Quality

- 如果用户要文件，必须真实生成 `.html` 与 `.docx`
- HTML 必须可直接阅读，不是终端文本堆叠
- Word 必须可打开，结构和标题层级清晰
- 表格列数一致，字段命名可复用
