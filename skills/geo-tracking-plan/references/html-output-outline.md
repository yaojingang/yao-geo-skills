# HTML Output Outline

## Goal

最终交付应能作为“给业务方或内部决策沟通使用的可视化方案页面”直接阅读，而不是终端结果堆叠。

## Required Sections

1. `封面摘要`
   - 公司名
   - 分析日期
   - 一句话结论
   - 当前追踪成熟度判断
2. `企业与当前 GEO 现状`
   - 企业简介
   - 核心产品与转化动作
   - 国内 / 海外 / 混合 GEO 判断
   - 当前承接资产与监测能力摘要
3. `证据面板`
   - 官网核验结果
   - 公开资料清单
   - 用户补充事实
4. `效果追踪方法与原理说明`
   - 品牌层价值与转化层价值拆分
   - 直接监测原理表
   - 间接监测原理表
   - 一组从 `AI 曝光 -> 承接 -> 归因 -> 复盘` 的流程图示
5. `直接效果追踪体系`
   - 每个监测动作的目标、口径、实现方式、优先级
6. `间接效果追踪体系`
   - 每个指标的意义、来源、更新频率、解释方式
7. `监测效果边界说明`
   - 可直接观测层
   - 可部分恢复层
   - 暂时不可观测层
   - `20%~30%` 作为保守规划假设的说明
8. `归因数据表设计`
   - 字段建议
   - 数据源
   - 主键或关联逻辑
9. `优先落地路线图`
   - 30 天
   - 60 天
   - 90 天
10. `风险与边界`
   - 无法直接追踪的部分
   - 置信度与资料缺口

## Suggested Visual Blocks

- 顶部 KPI 卡片
- 价值拆分卡片：品牌价值 / 直接效果 / 间接效果
- 监测方法说明表
- 分阶段流程图或链路图
- “直接 / 间接”双栏结构
- 证据来源表格
- 路线图时间轴
- 风险与缺口提示框

## Writing Rules

- 页面里重复公司名、品牌名、产品名，少用代词
- “最新”一律带绝对日期
- 事实和推断要明显分栏或分标签
- 没有真实数据时不要伪造图表数值
