# Attribution Framework

## Core Principle

GEO 后端归因不要假设“所有效果都能被直接追踪”。更现实的做法是把 GEO 价值拆成三个层面：

- `品牌层价值`
- `直接效果监测`
- `间接效果监测`

三者共同支持趋势判断、阶段复盘和优化，而不是追求虚假的全量归因。

## 1. Brand-Level Value

当用户在 AI 搜索或 AI 问答平台中搜索相关问题时，如果品牌被 AI 作为推荐对象、引用对象或对比对象展示，这本身就构成了高信任度的“第三方背书”。

### What It Changes

- 提升品牌被主动想起的概率
- 强化专业度、可信度和默认候选心智
- 缩短后续咨询、比较与成交阶段的信任建立时间
- 提高后续品牌词搜索、官网访问和咨询转化的效率

### Measurement Reminder

- 品牌层价值通常不能被单一后台直接计量
- 更适合通过品牌词、直接访问、咨询话术、自报来源和线索质量变化间接观察
- 品牌层价值通常与后续转化效率正相关，但不要写成“固定倍数”或“必然等比放大”

## 2. Direct Effect Monitoring

适合可以设计专属承接动作的企业。

### Common Patterns

- AI 专属电话、分机或手机号
- AI 专属企微号、顾问号、客服入口
- AI 专属福利口令、邀请码、体验码
- AI 专属活动页、报名页、预约页
- AI 专属落地页或轻量承接页
- 表单来源字段、渠道码、页面 ID、活动 ID
- 可识别的 URL 参数或 Web referer

### Principle Table

| 方法 | 监测原理 | 更适合的场景 | 关键执行动作 |
|---|---|---|---|
| Web referer / URL 参数 | 识别可回传的访问来源与转化路径 | 海外官网承接较强、网页链路较完整 | 接入分析平台、标准化来源参数、按 AI 平台聚合 |
| GEO 专属落地页 | 将 GEO 承接行为集中到专属页面 | 官网能力较强、可快速做页面分流 | 建专属页面、绑定 CTA、关联 landing_page_id |
| 表单隐藏字段 | 把渠道、平台、主题写入线索主键 | 注册、预约、试用类业务 | 新增 channel_group、ai_platform、keyword_theme 等字段 |
| 专属热线 / 分机 | 用电话号码或分机区分 GEO 线索 | 国内咨询、电话承接明显的业务 | 设置热线分机、记录通话结果、同步 CRM |
| 专属企微 / 顾问入口 | 用专属活码或顾问入口恢复来源 | 国内高意向咨询、社交承接明显的业务 | 配置活码 ID、顾问入口标签、回写 CRM |
| 福利口令 / 体验码 | 用用户主动报码完成来源识别 | referer 丢失严重、需要低成本补丁 | 设计易记口令、培训顾问追问、录入 CRM |
| 首问问卷 / 自报来源 | 用自报方式恢复来源 | 跨端跳转、官网外成交链路较长 | 在表单或顾问首问里加入来源问题 |
| 人工补录 | 用顾问或运营回填来源信息 | 长销售链路或多触点业务 | 设计补录字段、审核状态和补录原因 |

### Notes

- Referer 更适合网页端，App 内链路经常不完整
- 口令、权益码和问卷是应对来源丢失的低成本补丁
- 高客单业务优先把承接动作做到“顾问入口 + 表单字段 + 线索台账”

## 3. Indirect Effect Monitoring

适合无法稳定做强归因闭环，或需要补强趋势分析的企业。

### Common Metrics

- 品牌词检索量变化
- 官网整体 UV 变化
- GEO 专属落地页 UV 变化
- CTA 点击率变化
- 注册率、预约率、试用申请率变化
- 线索率、线索到商机率、成交率变化
- 注册后或购买后来源问卷

### Principle Table

| 指标 | 监测原理 | 更适合的场景 | 关键执行动作 |
|---|---|---|---|
| 品牌词检索量 | 观察 AI 曝光后是否带来主动品牌搜索 | 品牌认知建立、品牌心智明显受益的业务 | 固定品牌词包、按周比较趋势 |
| 官网整体 UV | 看 AI 曝光是否带来更多直接或自然访问 | 官网为主要承接阵地 | 对比活动前后与主题页变化 |
| GEO 页 UV / CTA 点击率 | 观察 GEO 专属承接页是否有效接住需求 | 有专属落地页或活动页 | 监测页面访问、CTA 点击与跳转 |
| 注册率 / 预约率 / 试用率 | 观察中间转化动作是否提升 | PLG、试用、预约类业务 | 固定分母分子口径并按主题页复盘 |
| 线索率 / 商机率 | 判断 GEO 是否带来更高质量线索 | B2B 销售型业务 | 对齐 CRM 阶段定义，按来源组复盘 |
| 成交率 / 升级率 | 判断 GEO 是否推动最终业务结果 | 有明确成交或升级动作的业务 | 与销售周期一起看，避免短期误判 |
| 自报 AI 来源占比 | 恢复没有 referer 的间接影响 | 国内跨端跳转和顾问转化场景 | 在问卷、顾问首问、售后回访中标准化提问 |

### Interpretation

- 间接指标不能单独证明 GEO 贡献
- 间接指标需要和关键词主题、页面变化、承接动作一起解释
- 长决策链业务里，成交率变化通常滞后于流量与线索变化

## 4. Measurement Boundary Statement

GEO 的总价值通常无法被后台完整显性统计。为了避免错误承诺，建议在交付中明确区分：

- `可直接观测层`
- `可部分恢复层`
- `暂时不可观测层`

### Conservative Planning Heuristic

- 很多企业在刚完成 GEO 监测体系部署时，可直接观测的转化通常只占总贡献的一部分
- 常见保守规划假设可以写成 `20%~30%`
- 海外官网承接强、表单字段完整、注册链路标准化的场景，可直接观测比例可能更高
- 国内多触点、跨端、顾问成交型业务，直接观测比例通常更低，对来源补丁依赖更强

### Important Guardrail

- `20%~30%` 只能作为解释边界和规划假设
- 不能写成所有品牌、所有行业都成立的普适事实
- 不要直接拿固定倍数推导总 ROI，更稳的做法是用直接观测值校准方向，再结合品牌词、咨询质量、商机率、成交效率和时间滞后一起判断

## 5. Output Requirements

方案里至少回答这几个问题：

- 品牌层价值主要体现在哪些信任与心智变化上
- 追什么
- 怎么追
- 从哪里取数
- 更新频次是什么
- 谁来维护
- 哪些数据能直接解释，哪些只能辅助判断

并且至少增加两个解释模块：

- `效果追踪方法与原理说明`
- `监测效果边界说明`

## 6. Market Split Reminder

- 海外 GEO 可以更多依赖官网页矩阵、表单字段、页面事件和升级路径
- 国内 GEO 通常要加强口令、电话、企微、问卷和人工补录
- 如果不先判断市场环境，就容易把同一套方法硬套到所有公司上
