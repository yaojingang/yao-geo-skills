# Attribution Framework

## Core Principle

GEO 后端归因不要假设“所有效果都能被直接追踪”。更现实的做法是拆成：

- `直接效果监测`
- `间接效果监测`

两者共同支持趋势判断、阶段复盘和优化，而不是追求虚假的全量归因。

## 1. Direct Effect Monitoring

适合可以设计专属承接动作的企业。

### Common Patterns

- AI 专属电话、分机或手机号
- AI 专属企微号、顾问号、客服入口
- AI 专属福利口令、邀请码、体验码
- AI 专属活动页、报名页、预约页
- AI 专属落地页或轻量承接页
- 表单来源字段、渠道码、页面 ID、活动 ID
- 可识别的 URL 参数或 Web referer

### Notes

- Referer 更适合网页端，App 内链路经常不完整
- 口令、权益码和问卷是应对来源丢失的低成本补丁
- 高客单业务优先把承接动作做到“顾问入口 + 表单字段 + 线索台账”

## 2. Indirect Effect Monitoring

适合无法稳定做强归因闭环，或需要补强趋势分析的企业。

### Common Metrics

- 品牌词检索量变化
- 官网整体 UV 变化
- GEO 专属落地页 UV 变化
- CTA 点击率变化
- 注册率、预约率、试用申请率变化
- 线索率、线索到商机率、成交率变化
- 注册后或购买后来源问卷

### Interpretation

- 间接指标不能单独证明 GEO 贡献
- 间接指标需要和关键词主题、页面变化、承接动作一起解释
- 长决策链业务里，成交率变化通常滞后于流量与线索变化

## 3. Output Requirements

方案里至少回答这几个问题：

- 追什么
- 怎么追
- 从哪里取数
- 更新频次是什么
- 谁来维护
- 哪些数据能直接解释，哪些只能辅助判断

## 4. Market Split Reminder

- 海外 GEO 可以更多依赖官网页矩阵、表单字段、页面事件和升级路径
- 国内 GEO 通常要加强口令、电话、企微、问卷和人工补录
- 如果不先判断市场环境，就容易把同一套方法硬套到所有公司上
