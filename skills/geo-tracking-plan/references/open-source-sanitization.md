# Open-Source Sanitization

## Goal

把一个原本服务内部项目的 skill 改造成可公开发布的版本时，重点不是“删掉几行字”，而是彻底移除对私有环境的依赖，并清理会泄露公司、团队和个人信息的样本。

## Must Remove

- 私有 CLI、私有 API、内网地址、个人账号路径
- 公司内部知识库、内部周报、内部复盘、CRM 截图
- 未公开词包、未公开项目进度、未公开转化数据
- examples、assets、reports 中残留的真实公司信息
- 本地输出文件里的绝对路径、用户名、系统名、时间戳水印

## Example Replacement Rules

- 用公开可理解的公司替换私有样本
- 最好使用 `公开公司 + 合成演示数据` 的方式，而不是照搬真实业务数据
- 示例里要显式写出 `illustrative`、`synthetic` 或同等说明，避免被误认为官方报告
- 如果要保留真实公司名，只能使用官网和公开资料可支持的内容，不能写入未公开指标

## Safe Demo Pattern

- 公司：公开公司，例如 `HubSpot`
- 数据：合成但合理的指标和行动建议
- 语气：方法演示，不冒充真实审计结果
- 输出：保留结构、字段和样式，去掉任何私有事实链

## Pre-Publish Checklist

- 检查 `SKILL.md` 是否还提到内部系统或团队流程
- 检查 `references/` 是否残留内部资料
- 检查 `scripts/` 是否写死私有路径或内网依赖
- 检查 `examples/` 是否含真实公司数据、截图、导出物
- 检查生成文件元数据是否含个人用户名或本机路径
