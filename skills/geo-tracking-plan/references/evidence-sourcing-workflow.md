# Evidence Sourcing Workflow

## Goal

在不依赖任何内部系统的前提下，为 GEO 后端归因方案收集足够的公开证据与用户输入，避免输出成为泛泛模板。权威性默认以官网和其他官方资产为锚点。

## Retrieval Order

1. `用户提供材料`
   - 公司名、品牌别名、产品名
   - 官网链接、业务介绍、已有落地页或表单
   - 当前 GEO 目标、希望追踪的动作、现有数据条件
2. `官方资产`
   - 官网首页
   - 关于我们 / 公司介绍
   - 产品页 / 解决方案页
   - 联系页 / 定价页 / Demo 申请页
   - 招聘页 / 法律页 / 社媒链接
3. `辅助公开资料`
   - 公开演讲、公开案例、帮助中心、开发者文档
   - 官方社媒、官方视频、公开 PR 或 release note
   - 可信第三方目录页或媒体稿，只能作为补充，不能替代官网

## Source Priority

默认按这个优先级取证：

1. 官网及官方资产
2. 用户提供的业务背景与现有承接信息
3. 官方帮助中心、文档、产品资源
4. 可信第三方公开资料

只要官网和官方资产已经足够支撑结论，就不要过度堆叠二手资料。

## Evidence Table Fields

每条证据至少记：

- `source_system`
- `source_locator`
- `title`
- `absolute_time`
- `entity`
- `topic`
- `key_fact`
- `fact_or_inference`
- `how_used`

## Official Site Validation

- 看品牌自称是否一致
- 看联系信息是否自洽
- 看法律页、招聘页、社媒链接是否回到同一品牌
- 看产品页、案例页、博客页是否共用主域名或官方子域名

## Freshness Rules

- “最新”必须有绝对日期
- 没有日期时，用“当前可见公开版本”替代“最新”
- 同一主题若有多份资料，优先保留最近日期且最贴近转化动作的版本

## Missing Data Handling

如果缺关键材料，明确写出：

- 已搜索范围
- 已拿到的资料类型
- 缺失的关键资料
- 方案因此降低确定性的部分
